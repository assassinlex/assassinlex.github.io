Docker初探

Docker
 2017/12/24

由于公司的服务器程序版本过于老旧，生产环境和本地开发环境的差异给开发带来很多不方便。为了环境的统一，可以采取Vagrant，或者Docker解决环境问题。下文介绍一些Docker的基础知识。

首先介绍一下什么是Docker，它能做些什么，如何来使用它等。

Docker简介

这里引用 Docker入门教程 中的定义：

Docker是一个开源的引擎，可以轻松的为任何应用创建一个轻量级的、可移植的、自给自足的容器。开发者在笔记本上编译测试通过的容器可以批量地在生产环境中部署，包括VMs（虚拟机）、bare metal、OpenStack 集群和其他的基础应用平台。

简单来说，Docker是基于Go语言实现的一个云开源项目，它能高效地构建应用程序，在部署应用程序这一个环节节省大量时间，使得开发人员不用花太多时间在环境部署上面。

Docker的核心概念

Docker的核心总的来说可以分为3个部分：镜像(images)、容器(containers)、仓库(repository)。

镜像

镜像可以理解为包含了一个操作系统的环境，比如一个Ubuntu/CentOS，那么可以将这个镜像称为Ubuntu/CentOS镜像。镜像也可以安装了某些应用程序，比如Apache/Nginx，那么我们也可以称它为Apache/Nginx镜像。

容器

容器类似于一个轻量级的沙箱，从镜像中创建的应用程序可以在容器中运行，可以启动、停止、删除容器，而不同的容器之间是相互隔离、相互不可见的。若把镜像看做一个包含有应用程序的操作系统，那么容器则是这些应用程序和操作系统运行的应用盒子。

仓库

仓库就比较好理解了。就好比你的代码仓库一样，你平时拉取、提交、推送时操作你的本地仓库和远程仓库。比如：你的本地git仓库存放着你的项目代码文件，而Docker仓库则是存放着镜像的文件。

拉取镜像

在拉取镜像之前，我们要设置一下Docker的镜像源，因为某些原因，直接访问国外镜像源的时候，网速会很慢，有时候会导致镜像下载不成功。打开docker的偏好设置选项， 选择Daemon标签，将适合你的镜像源地址添加到镜像源仓库地址列表中。

我使用的是阿里云的一个镜像地址，想用不同的同学可以去搜索一下，我这里推荐几个：

  • Docker官方中国区：https://registry.docker-cn.com
  • 网易：http://hub-mirror.c.163.com
  • 阿里云：https://22104bwc.mirror.aliyuncs.com
