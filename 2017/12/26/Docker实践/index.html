由于笔记本装的是最新版本的php7.2，导致一些老项目无法运行，今天就做了一个lamp的镜像，顺便学习了一下微软的SQL Server-for-Linux的镜像，这里就分享一下。

php需要特殊的远古版本5.3，且web服务器用的是apache而不是nginx，所以apache和php是在基础镜像的基础上自定义安装的，mysql是独自另外一个镜像。原本打算mysql也和php装在一个镜像中，这样每次只需要启动一个容器就可以了，但是由于技术不到家，当装完amp三件套后保存为新的镜像，启动容器时mysqld总是无法启动，进入shell查看原因时发现是无法更新进程文件导致的，但是我用以前的修改my.cnf配置文件中的pid-file选项和修改目录权限的操作后，依然无法解决这个问题，就使用独立mysql镜像来解决。

SQL Server for Linux

某些特殊的场景下，数据可能存储到多处，比如 mysql、sql server、oracle等等。那在不是windows操作系统上的同学可能在操作sql server的时候，就比较麻烦了，以前的话，可以跑一个windows的虚拟机来运行sql server，然后配置一下外部访问就可以了。但是微软在今年出了一个SQL Server 2017，使得其可以运行在Linux/Unix系统上了，而且已经有多个发行版本的Linux包，各位可以自行到SQL Server for Linux官方文档上去学习学习，我这里就说一下在MacOS下用Docker如何来运行。

提醒：官方要求Docker内存分配至少2G，但是这里推荐设置4G。

写在最后

记录一次粗糙的Docker操作，很没有掌握Dockerfile的使用，有很多容器服务安装和自启动是可以在Dockerfile里面完成的，且其能做到的远远不止这些，关于Docker的学习任重道远。
